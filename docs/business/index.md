# Бизнес-процесс

Этот раздел описывает бизнес-процессы ProfLaundry.

Подход: каждый процесс разбит на **минимальные атомарные части**, которые могут существовать независимо друг от друга. Это позволяет:
- использовать каждое утверждение как самостоятельную единицу проверки
- сверять правильность реализации на каждом следующем этапе (модели, архитектура, UI)
- вносить изменения точечно, не затрагивая смежные части
