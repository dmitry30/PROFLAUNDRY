# Архитектура

## Типы объектов метаданных

Структура данных каждой организации строится по модели 1С:Предприятие. Все объекты делятся на следующие типы:

### Справочники
Мастер-данные. Хранят постоянную или редко меняющуюся информацию: контрагенты, номенклатура, сотрудники, склады и т.д. Поддерживают иерархию и группировку.

### Документы
Фиксируют хозяйственные операции. Ключевое свойство — **проведение**: документ влияет на регистры только после проведения. Отмена проведения откатывает все изменения в регистрах.

### Журналы документов
Группировки документов для удобного просмотра в интерфейсе. Не хранят данных — только отображают выборку из нескольких типов документов.

### Перечисления
Фиксированные наборы значений, заданные на уровне конфигурации: статусы, типы, режимы. Не редактируются пользователем.

### Константы
Единичные значения уровня организации: основная валюта, название, реквизиты. Изменяются редко.

### Планы счетов
Иерархический список бухгалтерских счетов. Используется регистрами бухгалтерии для двойной записи.

### Планы видов характеристик
Определяют наборы дополнительных реквизитов для справочников и документов. Позволяют гибко расширять объекты без изменения схемы.

### Планы видов расчёта
Описывают алгоритмы расчёта (начисления, удержания) — используются регистрами расчёта, например для зарплаты.

### Регистры накопления
Хранят обороты и остатки: товары на складах, денежные остатки, взаиморасчёты с контрагентами. Два вида:
- **Остатки** — хранят текущее состояние (можно запросить остаток на дату)
- **Обороты** — хранят только движения (остаток не запрашивается)

### Регистры сведений
Хранят историю состояний: история цен, настройки во времени, курсы валют. Могут быть периодическими (значение актуально с даты установки).

### Регистры расчёта
Используются для многошаговых расчётов (зарплата, начисления). Поддерживают понятие периода действия и базового регистра.

### Регистры бухгалтерии
Реализуют двойную запись по плану счетов. Каждая проводка — дебет одного счёта и кредит другого.

### Отчёты
Аналитические срезы данных. Строятся на основе регистров и справочников. Могут иметь настраиваемые параметры и группировки.

### Обработки
Служебные операции: импорт/экспорт данных, пакетные изменения, утилиты администрирования.

### Бизнес-процессы и Задачи
Маршруты согласования и автоматизации. Задача — единица работы внутри бизнес-процесса, назначается исполнителю. *(Планируются к реализации позже.)*

## Проведение документов

Документ проходит через состояния:

```
Черновик → Проведён → (Отменён)
```

- **Черновик** — документ создан, данные не влияют на регистры
- **Проведён** — движения записаны в регистры, документ защищён от изменений
- **Отмена проведения** — движения откатываются, документ возвращается в черновик

## Изоляция данных и уровни владения

Каждый объект метаданных принадлежит одному из трёх уровней:

| Уровень | Кто владеет | Пример |
|---------|-------------|--------|
| **Платформа** | Администраторы платформы | Организации, метаданные конфигурации |
| **Организация** | Сотрудники организации | Контрагенты, документы, регистры |
| **Клиент** | Клиенты организации | Свои заказы, свои счета |

**Принцип изоляции:** организационные таблицы не ссылаются напрямую на платформенные сущности через FK. Связь между уровнями — через отдельные таблицы-связки (link tables). Это сохраняет изоляцию модулей и защищает данные.

## Структура модуля (Django-приложение)

Каждый модуль — отдельное Django-приложение. Внутри может быть организован как:

```
module_name/
  directories/   # Справочники модуля
  documents/     # Документы модуля
  registers/     # Регистры модуля
  reports/       # Отчёты модуля
```

Или плоско (для небольших модулей) — отдельными файлами в корне приложения.
