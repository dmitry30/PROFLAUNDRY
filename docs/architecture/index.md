# Архитектура

Этот раздел описывает техническую архитектуру системы PROFLAUNDRY.

**Язык раздела:** точные группы (Django-приложения), их связи, зависимости, границы ответственности. Не «что делает бизнес» — а «как это устроено технически».

Бизнес-логика и условия — в разделе [Бизнес-процесс](ref:business).

## Принципы

- **Ядро + модули** — ядро самодостаточно, модули расширяют без изменения ядра
- **Слабые связи** — компоненты взаимодействуют через чётко определённые интерфейсы
- **Multi-tenant** — организации изолированы на уровне данных
- **1С-подход** — справочники, документы, проведение, регистры
