# Терминология

Единый словарь терминов PROFLAUNDRY. Все участники проекта используют эти названия.

**Правило:** в документации и в общении — русские названия. В коде — английские эквиваленты (указаны в столбце «В коде»).

---

## Продукт

| Термин | Определение | В коде |
|--------|-------------|--------|
| **PROFLAUNDRY** | Полное название продукта | `proflaundry` |
| **pro** | Краткое название | — |
| **Платформа** | Вся система целиком, включая все организации и платформенный уровень | `platform` |

---

## Структура платформы

| Термин | Определение | В коде |
|--------|-------------|--------|
| **Организация** | Независимый тенант на платформе. Имеет своих клиентов, сотрудников, данные. Не видит другие организации. | `Organization` |
| **Платформенный уровень** | Слой выше всех организаций: управление тенантами, биллинг платформы, мета-таблицы. Недоступен самим организациям. | `platform` (app) |
| **Администратор платформы** | Управляет всеми организациями и модулями. Видит всё. | `PlatformAdmin` |
| **Администратор организации** | Управляет конкретной организацией: сотрудники, настройки. | `OrgAdmin` |

---

## Модули

| Термин | Определение | В коде |
|--------|-------------|--------|
| **Модуль** | Отдельная функциональная единица. Каждый модуль — Django-приложение. | Django app |
| **Базовые модули** | Включены по умолчанию для всех организаций. Образуют рабочий минимум. | `is_base: True` |
| **Дополнительные модули** | Выключены по умолчанию. Подключаются по решению администратора платформы. | `is_base: False` |
| **Ядро** | Совокупность базовых модулей: Клиенты, Номенклатура, Производство, Бухгалтерия, Коммуникация, Заказы. | — |

## 1С-концепции (адаптированные)

| Термин (рус.) | Определение | В коде |
|---------------|-------------|--------|
| **Справочник** | Эталонные данные: клиенты, номенклатура, сотрудники. Редко меняются, не проводятся. | `directories.py` |
| **Документ** | Бизнес-событие: заказ, расчётный лист, маршрутный лист. Имеет статус проведения. | `documents.py` |
| **Проведение** | Атомарная операция фиксации документа. После проведения документ защищён от изменений и генерирует записи в регистрах. | `posting.py` |
| **Регистр накопления** | Хранит текущие остатки и итоги. Обновляется при проведении документов. | `registers.py` |
| **Регистр сведений** | Хранит историю изменений по периодам. | `registers.py` |
| **Журнал** | Сводный список нескольких типов документов для навигации. | list view |
| **Отчёт** | Запрос по регистрам. Только чтение. | report view |
| **Обработка** | Фоновая или пакетная операция. | service / task |
