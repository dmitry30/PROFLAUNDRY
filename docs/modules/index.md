# Модули платформы

Модули — это Django-приложения в папке `modules/`, каждое из которых реализует конкретную бизнес-область. Все модели модулей наследуются от абстрактных базовых классов `platform_core`.

## Структура модуля

Каждый модуль имеет стандартную структуру:

```
modules/
  <module_name>/
    __init__.py
    apps.py              # AppConfig, регистрация auditlog
    models/
      __init__.py        # реэкспорт всех моделей
      <model>.py         # по одной модели в файле
    migrations/
      __init__.py
      0001_initial.py    # и далее
    admin.py             # регистрация на org_admin
```

## Включение модулей

Модули добавляются в `INSTALLED_APPS` в `config/settings/base.py`:

```python
'modules.counterparties',
'modules.nomenclature',
'modules.orders',
'modules.billing',
```

Для каждой организации список активных модулей хранится в `OrganizationModule`. Это позволит в будущем включать/отключать модули для конкретных арендаторов.

## Доступные модули

| Модуль | Метка | Описание | 1С-аналог |
|--------|-------|----------|-----------|
| [counterparties](counterparties.md) | `counterparties` | Клиенты и контрагенты | Контрагенты, Партнёры |
| [nomenclature](nomenclature.md) | `nomenclature` | Товары, услуги, цены | Номенклатура, Виды цен |
| [orders](orders.md) | `orders` | Заказы клиентов | Заказ клиента |
| [billing](billing.md) | `billing` | Счета и оплаты | Счёт на оплату, Поступление ДС |

## Межмодульные зависимости

```
counterparties   ←── nomenclature
      ↑                  ↑
      └──── orders ───────┘
               ↑
             billing
```

- `orders` зависит от `counterparties` (клиент, контрагент, договор) и `nomenclature` (товар/услуга)
- `billing` зависит от `counterparties`, `nomenclature` и `orders` (счёт создаётся на основе заказа)
