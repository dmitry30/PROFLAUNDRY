# Модуль: Контрагенты (`counterparties`)

Модуль управляет клиентами, контрагентами (юрлицами), физическими лицами, банковскими счетами и договорами.

**Ключевая концепция:** у одного `Client` (клиент — субъект работы через портал) может быть несколько `Counterparty` (юрлиц/ИП), через которые проводятся сделки. Это соответствует модели «Партнёр → Контрагенты» в 1С:УТ.

---

## Модели

### `Individual` — Физическое лицо

**1С-аналог:** справочник «Физические лица»

Хранит персональные данные человека отдельно от его роли контрагента. Одно физлицо может быть связано с несколькими контрагентами (например, ИП и физлицо одновременно).

| Поле | Тип | Описание |
|------|-----|----------|
| `last_name` | CharField(100) | Фамилия |
| `first_name` | CharField(100) | Имя |
| `patronymic` | CharField(100) | Отчество (необязательно) |
| `birth_date` | DateField | Дата рождения |
| `inn` | CharField(12) | ИНН физлица |
| `snils` | CharField(14) | СНИЛС |
| `phone` | CharField(20) | Телефон |
| `email` | EmailField | E-mail |
| `is_deleted` | BooleanField | Пометка удаления |

**Свойство:** `full_name` — возвращает «Фамилия Имя Отчество».

---

### `Counterparty` — Контрагент

**1С-аналог:** справочник «Контрагенты»

Наследуется от `CatalogItem` (справочник с группами, иерархией, кодом, пометкой удаления).

Представляет юрлицо, ИП или физлицо, с которым организация ведёт взаиморасчёты.

| Поле | Тип | Описание |
|------|-----|----------|
| `counterparty_type` | CharField | Тип: `legal` / `ie` / `individual` |
| `inn` | CharField(12) | ИНН |
| `kpp` | CharField(9) | КПП (для юрлиц) |
| `ogrn` | CharField(15) | ОГРН или ОГРНИП |
| `legal_address` | TextField | Юридический адрес |
| `actual_address` | TextField | Фактический адрес |
| `phone` | CharField(20) | Телефон |
| `email` | EmailField | E-mail |
| `individual` | FK → Individual | Связь с физлицом (для типа `individual`) |

**Типы контрагента:**
- `legal` — Юридическое лицо
- `ie` — Индивидуальный предприниматель
- `individual` — Физическое лицо

---

### `Client` — Клиент

**1С-аналог:** справочник «Партнёры» в 1С:УТ

Наследуется от `CatalogItem`. Субъект взаимодействия с организацией через клиентский портал. Один клиент владеет несколькими контрагентами.

| Поле | Тип | Описание |
|------|-----|----------|
| `counterparties` | M2M → Counterparty | Все контрагенты клиента |
| `main_counterparty` | FK → Counterparty | Основное юрлицо (по умолчанию) |
| `phone` | CharField(20) | Телефон клиента |
| `email` | EmailField | E-mail клиента |

---

### `BankAccount` — Банковский счёт

**1С-аналог:** справочник «Банковские счета контрагентов»

Наследуется от `OrgScopedModel`. Хранит реквизиты расчётного счёта контрагента.

| Поле | Тип | Описание |
|------|-----|----------|
| `counterparty` | FK → Counterparty | Владелец счёта |
| `bank` | FK → Bank | Банк (из platform_core) |
| `account_number` | CharField(20) | Расчётный счёт (20 цифр) |
| `correspondent_account` | CharField(20) | Корреспондентский счёт |
| `currency` | FK → Currency | Валюта счёта |
| `is_main` | BooleanField | Основной счёт контрагента |
| `is_deleted` | BooleanField | Пометка удаления |

---

### `Contract` — Договор

**1С-аналог:** справочник «Договоры контрагентов»

Наследуется от `CatalogItem`. Используется как аналитический разрез в документах расчётов.

| Поле | Тип | Описание |
|------|-----|----------|
| `counterparty` | FK → Counterparty | Контрагент по договору |
| `contract_type` | CharField | Вид: `buyer` / `supplier` / `other` |
| `number` | CharField(100) | Номер договора |
| `date_start` | DateField | Дата начала |
| `date_end` | DateField | Дата окончания |
| `currency` | FK → Currency | Валюта договора |

**Виды договора:**
- `buyer` — С покупателем
- `supplier` — С поставщиком
- `other` — Прочее

---

## Схема связей

```
Individual
    ↑ (ForeignKey, необязательно)
Counterparty ←── BankAccount
    ↑
Client (M2M) — main_counterparty (FK)
    ↑
Contract (FK к Counterparty)
```
